## 1. Fix Telegram Bots Not Responding

**Symptom:** User couldn't reach any agents on Telegram. All 3 bots (@mm11homebot,
@mhivepolybot, @mmhivepublisherbot) silent. Gateway healthy, containers running.

**Root cause:** mhive was asked to "turn everything off" to stop a runaway bill. mhive
correctly identified Telegram as the cost source but used the wrong levers:

1. `channels.telegram.enabled` set to `false`
2. `plugins.entries.telegram.enabled` set to `false`

The plugin being disabled was the actual blocker — gateway startup produces NO log lines
for Telegram when the plugin is off (silent failure). The channel flag alone would have
been insufficient too.

**Key discovery — Telegram has two independent switches:**

- `channels.telegram.enabled` — channel config layer
- `plugins.entries.telegram.enabled` — plugin registry layer
  Both must be `true` for bots to connect. Either one being `false` silently prevents all
  bots from starting.

**Also fixed:** Config had stale single-account top-level fields (`botToken: ""`,
`dmPolicy`, etc.) left over from a migration. Ran `openclaw doctor --fix` to clean up
(added `accounts.default`, renamed credential files). Removed stale `@openclaw/telegram`
plugin entry that was added by a failed `openclaw plugins enable` call.

**Fix applied:**

- `channels.telegram.enabled = true`
- `plugins.entries.telegram.enabled = true`
- Removed spurious `@openclaw/telegram` key from `plugins.entries`
- Ran `doctor --fix` for migration cleanup
- Restarted gateway → all 3 bots confirmed started in logs

## 2. Emergency Cost Kill Switch

**Problem framing:** mhive's instinct (disable Telegram) was understandable but wrong —
disabling channels cuts communication, requiring SSH to fix. The user needed a way to tell
mhive to stop spending WITHOUT losing the ability to talk to mhive.

**What actually costs money:** Heartbeat polling (every 30m per agent) and cron jobs
(scheduled tasks). NOT reactive responses to a single inbound message.

**Kill switch design — two levers only:**

| Lever             | Config key                        | Kill    | Resume  |
| ----------------- | --------------------------------- | ------- | ------- |
| Cron jobs         | `cron.enabled`                    | `false` | `true`  |
| Heartbeat polling | `agents.defaults.heartbeat.every` | `""`    | `"30m"` |

Telegram channels and plugins are NOT touched. mhive remains reachable throughout.
Gateway auto-restarts after `config.patch` (~5s). mhive confirms via Telegram.

**Deployed to:**

- All 14 TOOLS.md files on VPS (1 shared + 13 per-agent) — EMERGENCY STOP section appended
- `mhive-ops/ARCHITECTURE.md` — Emergency Procedures section added
- `MEMORY.md` (this machine) — kill switch pattern saved
- `TODO.md` — completion entries + notes added
- Staging synced via `sync-workspace.sh`

**Trigger phrases:**

- Kill: _"Emergency stop — kill the spending"_ or _"Activate kill switch"_
- Resume: _"Resume normal operations"_ (always confirm with Mehdi first)
