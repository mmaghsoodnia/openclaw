## Scout Diagnosis

Analyzed hive-scout as an example of a persistently broken agent. Found 6 compounding root causes:

1. **Model churn destroyed memory coherence** — Scout went through 4+ models (unknown → Grok → Gemini Flash → Grok Fast). Each left garbled memory entries (e.g., `memory/2026-02-23.md`: "I will have as it will be to do and make new log as req"). New models read old garbage and got confused.
2. **Models too weak for the task** — Gemini Flash scores 1.3/5 on quality benchmarks. Grok Fast scores 3.0/5. Both hallucinated market data instead of faithfully running the script and reporting.
3. **No RUNBOOK until March 7** — First 2+ weeks with zero operational procedures.
4. **Broken venv paths** — `source venv/bin/activate` fails in cron/exec contexts.
5. **Memory file naming chaos** — Models overwrote old memory files instead of creating new ones (e.g., `2026-02-24-market-scan.md` contains March 13 data).
6. **Inter-agent communication failures** — `sessions_send` broke on Mar 5 and Mar 8.

### Key Insight: Scout doesn't need to be an agent

Scout's entire job: run `fetch-soccer-markets.py` → filter → forward results to Percy.
No judgment, no reasoning, no adaptive behavior. An LLM agent wrapping a deterministic script
added failure modes (hallucination, memory corruption, model cost) without adding value.
This should be a cron job, not an agent.

## Mhive Programming v2 — Structural Hardening

After the initial health monitoring deployment, identified 6 structural risks and fixed all of them:

### 1. Health monitoring moved to dedicated cron job (from heartbeat)

- **Problem:** SOUL.md says use Grok-Fast for heartbeats, but health monitoring is complex reasoning work
- **Fix:** Created `mhive-health-check` cron job in `~/.openclaw/cron/jobs.json` with `model: anthropic/claude-sonnet-4-6` override
- Schedule: `0 10 */2 * *` (10 AM Pacific every 2 days)
- HEARTBEAT.md updated to remove Priority 0 (health monitoring), note cron handles it
- Cron enabled in `openclaw.json` (`cron.enabled: true`)

### 2. Closed-loop verification

- **Problem:** Mhive could fix things but never checked if fixes worked
- **Fix:** HEARTBEAT.md + cron job message now include "verify previous fixes" step. HEALTH-LOG.md tracks issue → action → verification result

### 3. Self-accountability (HEALTH-LOG.md)

- **Problem:** No way to know if mhive is actually running health checks
- **Fix:** Created `workspace/HEALTH-LOG.md` — running log with date, issues, actions, verifications. MM can glance at it to see if mhive is doing its job

### 4. Escalation via Telegram

- **Problem:** Writing to RESTRUCTURE.md could go unread
- **Fix:** HEARTBEAT.md rule: "After writing RESTRUCTURE.md, ALSO message MM on Telegram with one-line summary"

### 5. MEMORY.md trimmed (tactical → durable)

- **Problem:** Hive Status section got stale, polluted every message with outdated tactical info
- **Fix:** Removed Hive Status section. Tactical status now lives in HEALTH.md (generated by health script). MEMORY.md focused on: partnership, architecture patterns, coaching playbook, lessons learned, hard rules

### 6. HEURISTICS.md reminder

- **Problem:** HEURISTICS.md not in auto-injected workspace files, mhive might skip it before model changes
- **Fix:** Added "Before ANY model change: read HEURISTICS.md first" to TOOLS.md, HEARTBEAT.md, MEMORY.md Hard Rules, and cron job message
