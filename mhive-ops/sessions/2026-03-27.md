### Context Management Config (VPS)

Investigated openclaw's existing context management features. Found mature compaction, pruning, batching, and isolated session support already in the codebase.

**Config changes applied to VPS `openclaw.json`:**

- `agents.defaults.compaction.maxHistoryShare`: 0.5 → **0.35** (forces earlier compaction, more room for reasoning)
- `agents.defaults.compaction.keepRecentTokens`: unset → **30000** (preserves last ~30K tokens verbatim)
- `agents.defaults.compaction.recentTurnsPreserve`: unset → **3** (keeps last 3 full turns intact)
- `messages.queue.mode`: unset → **"collect"** (batches incoming messages)
- `messages.queue.debounceMs`: unset → **5000** (5s debounce window for batching)

**Rationale:** mhive's main session was at 432k tokens (43% of 1M). With `maxHistoryShare: 0.5` (default), compaction wouldn't trigger until 500K. Lowering to 0.35 means compaction triggers at 350K, keeping more headroom for reasoning. Message batching prevents the overload pattern we created by sending 3 coaching messages in rapid succession.

#### Key lesson: docker-compose.override.yml explicit env mapping

VPS `docker-compose.override.yml` explicitly lists each env var to pass to containers. Adding a var to `.env` alone is NOT enough — it must also be added to the override's `environment:` section for both services.
