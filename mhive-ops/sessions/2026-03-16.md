## Plugin Runtime Regression — Found and Fixed

After the upstream merge, all agent turns (both cron and interactive Telegram messages) failed with:

```
Unable to resolve plugin runtime module
```

### Root cause

`resolvePluginRuntimeModulePath()` in `src/plugins/loader.ts:223` looks for the plugin runtime at:

- `{packageRoot}/src/plugins/runtime/index.ts` (dev mode)
- `{packageRoot}/dist/plugins/runtime/index.js` (production/Docker)

But `tsdown.config.ts` never listed `src/plugins/runtime/index.ts` as a separate build entry. The runtime was bundled into the main entry chunk, so no standalone `dist/plugins/runtime/index.js` was emitted. This worked in dev (source tree available) but broke in Docker/production (only `dist/` exists).

Introduced by upstream plugin refactor commits:

- `9ab7fdfd03` Extension host: establish host-owned plugin runtime foundations
- `4c8853122a` Plugins: preserve lazy runtime provider resolution

### Fix

Added `src/plugins/runtime/index.ts` as a separate entry in `tsdown.config.ts`:

```typescript
nodeBuildConfig({
  entry: { "plugins/runtime/index": "src/plugins/runtime/index.ts" },
}),
```

This emits `dist/plugins/runtime/index.js` (a small re-export chunk pointing to shared bundles), which `resolvePluginRuntimeModulePath` can now find.

- Staging: zero errors, staging bot started clean

## BookHive Telegram Group Fix

Bookworm bot (@mmhivepublisherbot) was not responding to messages in the "Book group" Telegram group.

### Root cause

Mhive had added the group chat ID (`-5119732747`) to the bookworm account's `allowFrom` array. But `allowFrom` only accepts **user IDs** — group chat IDs are silently rejected with:

```
Invalid allowFrom entry: "-5119732747" - allowFrom/groupAllowFrom authorization expects
numeric Telegram sender user IDs only.
```

Additionally, without a `groups` config entry, the bot defaults to `requireMention: true` in groups, meaning it ignores messages unless @mentioned.

### Fix

1. Removed `-5119732747` from `allowFrom` (only user ID `217834570` remains)
2. Added `groups` config with `requireMention: false`:

```json
"bookworm": {
  "groups": {
    "-5119732747": { "requireMention": false }
  }
}
```

3. Restarted gateway — bookworm now responds to all messages in the Book group.

### Prevention

Added "Telegram Group Configuration" section to mhive's shared `TOOLS.md` documenting the correct way to configure groups (use `groups` config, not `allowFrom`).
