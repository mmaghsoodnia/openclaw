### 2. Staging gog Binary Fix

- **Root cause:** `~/.openclaw-staging/bin/gog` was a macOS Mach-O arm64 binary. Docker Desktop runs Linux aarch64 containers — binary can't execute (exec format error).
- **Fix:** Cloned `steipete/gogcli` from GitHub, cross-compiled for linux/arm64 using Docker Go container:
  ```
  docker run --rm -v /tmp/gogcli:/src -v /tmp/gog-out:/out -w /src golang:1.24 \
    sh -c "GOTOOLCHAIN=auto GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -o /out/gog ./cmd/gog"
  ```
- Replaced staging binary, restarted gateway container
- Verified: `docker exec ... gog --version` returns `gog version 0.12.0-dev`

### 6. Staging gog Credentials from 1Password (main deliverable #2)

- **Root cause of staging gog failure:** `docker-compose.staging.yml` mounted `~/.config/gogcli/` from Mac host — but that directory was always empty (macOS uses `~/Library/Application Support/gogcli/`). Staging gog was never working, even before the keychain switch.
- **Design principle:** All credentials from 1Password. Both VPS and staging Docker environments get secrets injected from 1Password — nothing lives only on disk.
- **What was done:**
  - Created `mhive-ops/staging/setup-gog.sh` — reconstructs `~/.openclaw-staging/gogcli/` from 1Password: static config.json, credentials.json via `op inject` template, keyring tokens via `op read` + base64 decode
  - Created `mhive-ops/staging/.gogcli-credentials.tpl` — `op inject` template for OAuth client_id + client_secret
  - Updated `docker-compose.staging.yml` — mount changed from `~/.config/gogcli/` to `~/.openclaw-staging/gogcli/`
  - Integrated `setup-gog.sh` into `start.sh` (step 3/7) and `rebuild.sh`
  - Updated `ARCHITECTURE.md` — new "Google Workspace (gog) credentials" section in Secrets Management

### 8. Staging gog End-to-End Test

- Ran `setup-gog.sh` — reconstructed `~/.openclaw-staging/gogcli/` from 1Password
- **Bug found:** `op inject` embedded trailing newlines inside JSON string values in `credentials.json`. `gog` failed with `decode credentials: invalid character '\n' in string literal`.
- **Fix:** Replaced `op inject` with `op read` + `printf` + `tr -d '\n\r'` in `setup-gog.sh`. Removed the now-unused `.gogcli-credentials.tpl` template file.
- Restarted staging gateway, verified all three Google APIs from inside Docker container:
